# Quiet mode for CI/CD pipelines

## Description

Add a `quiet` boolean to config (and a corresponding `PARTIO_QUIET` env var) that suppresses all stdout/stderr output from hook scripts in CI environments. When quiet mode is active, the hook runner should redirect slog output to discard and skip any `fmt.Print*` calls. The hook shell scripts generated by `internal/git/hooks/hooks.go` should also suppress output by redirecting to `/dev/null` when the env var is set.

## Why

CI pipelines treat unexpected stdout/stderr from hooks as noise or errors. Currently partio's warning and debug logs can appear in CI output, confusing build logs. This is a low-effort quality-of-life change that makes partio production-friendly.

## User Relevance

Teams running partio in CI (e.g. to capture agent-assisted automated commits) can enable quiet mode to keep build logs clean without disabling partio entirely.
